HTML - HyperText Markup Language

  • Le HTML est un langage de markup (il décrit les éléments présents), pas de programmation.
  • Actuellement, en version 5 (évolue toujours, la V5 date de 2014).
  • Un document HTML est composé de balises (ou tags), délimitées par des chevrons (< >). Les balises peuvent contenir du texte et / ou d'autres balises.

Structure d'un document HTML

  • Déclaration de la version HTML utilisée
    (<!DOCTYPE html> pour HTML 5)
  • En-tête (head)
  • Corps du document (body)
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8"/>
        <title>Première page web</title>
    </head>
    <body>
        <p>Ceci sera affiché dans le navigateur</p>
    </body>
</html>

Deux types de balises

  • Balises qui délimitent une zone
    • Balise de début de zone <p>
    • Balise fermante ou de fin de zone </p>
    <p>
    La balise p délimite un paragraphe 
    de texte
</p>
  • Balises autonomes : se suffisent à elles-mêmes et n'ont pas besoin d'être fermées. Par exemple, <br> insère un saut de ligne (avant HTML5, il fallait finir ces balises par '/', vous verrez donc souvent <br/>)

Règles de balisage

Les balises peuvent être imbriquées :

texte en italique et en gras

Une balise doit toujours se fermer dans la même balise que celle où elle a été ouverte, le code suivant n'est donc pas correct :

Ceci n'est <em>pas <strong>correct</em> du tout</strong>

Inline et Block

On distingue :

  • les balises bloc, qui peuvent contenir des balises en-ligne et en bloc (sauf exceptions)
    Exemples: p, h1, ul, table, form, pre, div
  • les balises en-ligne (inline), qui ne peuvent contenir que d'autres balises en-ligne (sauf exceptions)
    Exemples: em, strong, img, a, span

Les attributs

Les balises peuvent avoir des attributs, qui vont préciser la manière dont elles se comportent.
Ex : la balise autonome <img/>, qui insère une image, a besoin du chemin vers l'image en question, fourni par l'attribut "src" : 

<img src="chat.gif" />

La valeur de l'attribut doit être entre guillemets

Commentaires

En HTML, un commentaire se fait avec la syntaxe suivante : 

<p> début du paragraphe
<!-- Tout ce qu'il y a ici est un commentaire et 
n'apparaitra pas dans le document -->
fin du paragraphe </p>

Caractères spéciaux

Les caractères utilisés pour le HTML, tels que <, > et &, doivent être encodés avec si on veut les afficher tels quels.

  • < s'encode &lt; (lt pour less than)
  • > s'encode &gt; (gt pour greater than)
  • & s'encode &amp; (ce caractère se nomme ampersand en anglais, et esperluette en français)

Il faut aussi encoder les caractères accentués / non ASCII si on n'utilise pas l'UTF 8 => toujours inclure une balise <meta charset="utf-8"> dans head

Espacement

Dans un document HTML, les sauts de ligne et espaces (après le premier) à l'intérieur des balises sont ignorés. Pour utiliser un texte préformaté avec les espaces tels quels, on peut utiliser une balise <pre>...</pre>.

Pour sauter une ligne à l'intérieur d'un bloc, on utilise la balise <br>.

Le contenu de la balise head

La balise <head> est l'en-tête du fichier HTML. C'est là qu'on spécifie le titre (obligatoire) de la page, qu'on inclut des fichiers de scripts et de style, et qu'on donne des informations sur la page. 

<head>
	<meta charset="UTF-8">
	<meta author="Timothée Behra">
	<title>Le header</title>
	<link rel="stylesheet" type="text/css" href="css/style.css">
	<script src="script.js" type="text/javascript"></script>
</head>

La balise body

  • Contient l'intégralité de la page elle-même
  • Avec <head>, c'est la seule autre balise que peut contenir la balise <html>

Les liens hypertextes

  • un lien est un élément cliquable permettant d'atteindre un document désigné par une URL
  • on utilise la balise en-ligne <a>texte du lien</a> et on spécifie l'URL avec l'attribut href (pour hypertext reference)
  • peut être absolu, relatif, ou interne à un document

Les images

  • on insère une image avec la balise <img /> (une balise autonome et en-ligne)
  • attributs :
    • src (url de l'image)
    • alt (légende / texte qui remplace l'image si elle ne s'affiche pas)
    • height (hauteur)
    • width (largeur)

Les listes

  • Pour faire une liste, on ouvre une balise <ul> (liste non ordonnée, à puces) ou <ol> (liste ordonnée), puis chacun des éléments est dans une balise <li>
  • Attribut type, peut prendre une des valeurs"disc|circle|square" pour ul, ou "1|I|i|A|a" pour ol
  • On peut créer des listes imbriquées

Balises génériques : div et span

  • <div> permet de définir une balise de type block générique
  • <span> permet de définir une balise de type inline générique
  • ces balises génériques ne devraient être utilisées que lorsqu'aucune autre balise sémantique n'est appropriée

Deux attributs universaux

  • L'attribut id permet de donner un identifiant à une balise unique. La présence de plusieurs éléments avec un même id cause une erreur
  • L'attribut class permet de décréter que la balise appartient à une ou plusieurs classes d'éléments
  • Ces 2 attributs peuvent être donnés à n'importe quelle balise, et servent à identifier ces éléments pour leur donner un style ou leur appliquer une logique

Les tableaux

  • La balise <table> permet de définir un tableau à deux dimensions
  • On commence chaque ligne par une balise <tr> (pour table row)
  • Une ligne <tr> contient ensuite des balises <td> (cellule normale) et <th> (cellule de titre).
    Les <td> et <th> contiennent le contenu des cellules
  • Autres éléments pouvant être contenus dans une <table> : <caption>, <thead>, <tbody>, <tfoot>

Exercice

  • Écrivez un document en pur HTML qui contient un tableau avec au moins 3 lignes dont une ligne de titre et au moins 4 colonnes.
  • La première colonne doit comprendre une image qui, lorsqu'on clique dessus, renvoie sur le site où cette image a été trouvée.
  • Toutes les lignes doivent avoir la même hauteur.
  • Au moins une des cases contient une liste d'éléments
  • Contenu au choix - si pas d'inspiration, informations sur des animaux (cf exemple)

La balise form

  • Déclaration du formulaire : <form> ... </form>
  • Attributs importants :
    • action : URL vers laquelle envoyer les données saisies
    • method : commande http (get ou post) à utiliser pour effectuer l'envoi
  • Toutes les balises sont permises dans un formulaire

La balise input

  • Balise en ligne, autonome, qui doit être déclarée dans unformulaire
  • name : le nom du champ de saisie, unique dans le formulaire, permet de récupérer sa valeur
  • type :
    • text : zone de texte (type par défaut)
    • radio : un choix maximum parmis plusieurs
    • checkbox : chaque case peut être cochée ou non
    • number : ne permet de rentrer qu'un nombre

La balise label

  • Un label est un texte associé à un champ de formulaire. Un clic sur le label met le focus sur le champ
  • Il faut utiliser l'attribut for et lui donner l'id du champ correspondant (attention, pas le name)

Envoi des données au serveur

  • Lorsque l'utilisateur appuie sur le bouton submit, le navigateur construit une chaîne de caractères contenant toutes les données du formulaire, et l'envoie au serveur.
  • Ensemble de couples séparées par le caractère &. Chaque couple est de la forme nom de champ = valeur saisie. Les espaces sont remplacés par le caractère + (ou %20). Les caractères + & = sont encodés %2B %26 %3D

Envoi des données au serveur

  • Si le formulaire utilise la méthode "get", la chaine de caractère représentant les réponses au formulaire est visible dans l'URL de la page désignée par l'attribut action
