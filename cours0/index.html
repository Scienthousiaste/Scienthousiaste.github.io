Etat d’esprit du cours

But : Vous faire plaisir avec un projet qui vous inspire.

Contraintes :

  • Travailler toutes les technologies vues en cours
  • L'objectif est de maîtriser les bases, donc : pas de CMS, frameworks ou de librairies CSS

Honnêteté :

  • Réutiliser du code trouvé sur internet = OK
  • Ne pas le comprendre intégralement = pas OK du tout

Librairies

Caractéristiques :

  • But : Facilite le développement
  • Utiles pour les petits et gros projets
  • Rapidement utilisables

Exemples :

  • jQuery permet de simplifier le Javascript
  • Bootsrap permet de simplifier la mise en forme (et plus)

Frameworks

Caractéristiques :

  • But : change la manière donc le code est écrit et structuré
  • Plus long à mettre en place et à apprendre
  • Utiles pour les gros projets

Exemples :

  • Django : framework web pour python
  • Angular2 : framework web pour javascript

Mais dans le privé...

  • ...Utilisation massive de librairies, frameworks, CMS
  • Alors pourquoi pas utilisation dès maintenant ?
  • Car éloignement des fondamentaux

Mais dans le privé...

Il y a une différence entre :

  • coder pour apprendre
  • coder pour développer un projet efficacement

Recommandations :

  • Projets perso
    => utiliser des librairies
  • Chercher un poste de développeur web
    => Apprendre un framework backend et/ou frontend pour sortir du lot
